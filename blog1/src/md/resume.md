## Projects


### chatGraph (AI 대화 시각화 플랫폼)

**서비스 개요**: AI와의 대화를 꼬리물기 형식의 마인드맵 그래프로 시각화하여 사고 확장을 돕는 웹 서비스

**진행기간: 2025.04 ~ 2025.12**

**팀원:** 4명 (FE 2명, BE 2명)

기술 스택: TypeScript, React, Next.js, D3.js, Zustand, Tailwind CSS, TanStack Query

- 선형적인 텍스트 UI에서 발생하는 대화 흐름 파악의 어려움을 해결하고자 D3.js Force Layout을 적용하여 비선형적 대화 구조를 시각화.
- 질문 트리의 깊이가 증가함에 따른 렌더링 저하 문제를 해결하기 위해 재귀적 상태 관리와 부분 업데이트 최적화를 적용하여 10 depth 이상의 구조에서도 60fps의 안정적인 성능을 확보.
- 프로젝트 규모 확장에 대비하여 Feature-First Architecture를 도입하고, 기술적 관심사(UI)와 도메인 관심사(Feature)를 분리하여 순환 참조 문제를 해결하고 유지보수성을 극대화.
- 단일 컴포넌트에 과도하게 집중된 책임을 분산시키기 위해 'God Component'를 기능별로 분해(Decomposition)하여 코드 복잡도를 낮추고 핵심 로직 라인 수를 340줄에서 45줄로 개선.
- 질문 추가 시 발생하는 서버 응답 지연 문제를 해결하기 위해 Temp ID 기반의 낙관적 업데이트(Optimistic Update)를 도입하여 사용자에게 즉각적인 반응 경험을 제공.
- React Suspense와 Streaming SSR을 효과적으로 지원하기 위해 ChatView를 Standard/Optimistic 모드로 분리하고, 비동기 데이터 로딩 중에도 끊김 없는 UX를 구현.
- 뷰(View) 계층의 코드 중복과 역할 모호성을 해결하기 위해, 렌더링 책임을 전담하는 Layout 컴포넌트를 분리하고 View Composition 패턴을 적용하여 DRY 원칙을 준수.
- 복잡한 비동기 상태 관리로 인한 보일러플레이트 코드를 제거하고자 TanStack Query를 도입하여 데이터 캐싱과 서버 상태 동기화를 자동화.
- D3.js의 직접적인 DOM 조작과 React의 가상 DOM 렌더링 충돌을 방지하기 위해 라이프사이클을 세밀하게 제어하여 React 생태계 내에서 동작하는 안정적인 그래프 차트를 구현.
- 프론트엔드의 트리 구조와 백엔드 RDB 구조 간의 불일치를 해결하기 위해 재귀적 JSON 직렬화 규격을 설계하여 데이터 통신 효율성을 최적화.

### FiT (사일런트 컨퍼런스 플랫폼)

**서비스 개요:** 웹으로 즐기는 실시간 사일런트 컨퍼런스 오디오 스트리밍 플랫폼

**진행기간: 2025.02~2025.04**

**팀원:** 4명 (FE 2명, BE 4명, PD 2명, PM 1명)

**기술 스택:** React, Vite, WebRTC, STOMP/SockJS

- 사일런트 컨퍼런스 오디오 송출 시 네트워크 지연으로 인한 끊김 현상을 해결하기 위해 WebRTC와 STOMP 시그널링을 적용하여 0.5초 미만의 초저지연 스트리밍 환경을 구축.
- 컨퍼런스 정보 전달 과정에서 긴 텍스트로 인한 사용자 이탈을 방지하고자 Scroll-telling 인터랙션을 도입하여 사용자 UX 증대.
- Framer Motion 라이브러리를 도입하고 물리 엔진 기반의 자연스러운 애니메이션을 적용하여 사용자의 서비스 몰입감을 극대화.
- Tailwind CSS의 JIT 컴파일러와 유틸리티 클래스를 적극 활용하여 CSS 번들 사이즈를 60% 절감하고 렌더링 성능을 최적화.
- 백엔드 팀과의 원활한 협업을 위해 WebRTC 시그널링을 위한 STOMP 프로토콜 및 WebSocket 통신 과정을 이해하고 연결 로직을 함께 설계.

### xAB (A/B 테스트 기반 SNS)

**서비스 개요:** 두 가지 선택지에 대한 실시간 투표 및 댓글 토론이 가능한 소셜 네트워크 서비스

**진행기간: 2024.12~2025.01**

**팀원:** 4명 (FE 4명)

**기술 스택:** TypeScript, Next.js, React Query, Zustand, Supabase, Tailwind CSS

- 대량의 피드 이미지 로딩 시 발생하는 2.5초 이상의 LCP 지연 문제를 해결하기 위해 TanStack Query 기반의 무한 스크롤 및 지연 로딩을 적용하여 초기 로딩 속도를 1.2초로 단축.
- 실시간 댓글 및 투표 상호작용 과정에서 발생하는 UI 멈춤 현상을 방지하고자 낙관적 업데이트(Optimistic Update)를 적용하여 네트워크 지연 없는 즉각적인 반응성을 확보.
- 게시글 상세 페이지 진입 시 잦은 이동으로 인한 맥락 끊김을 해결하기 위해 Next.js의 병렬 및 인터셉팅 라우트 기반 모달을 구현하여 피드 위치를 유지하는 끊김 없는 탐색 경험을 제공.
- 투표 결과 및 유저 세션 정보의 정합성을 보장하기 위해 Zustand 기반의 전역 상태 관리 시스템을 구축.
- Supabase Realtime을 연동하여 댓글 및 투표 수치를 실시간으로 동기화하고, 디바운싱(Debounce) 로직을 통해 중복 요청을 방지.
- 사용자별 데이터 접근 제어를 위해 Supabase의 RLS 정책을 학습하고 기본적인 보안 규칙을 백엔드 수준에서 직접 설정.
